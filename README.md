1.-¿Qué es GitHub?, ¿Qué relacion tiene con Git?
GitHub es una plataforma en línea que permite alojar, compartir y colaborar en proyectos de software. Se usa principalmente para guardar código y trabajar en equipo.

La relación entre Git y GitHub es la siguiente:

Git es un sistema de control de versiones.
Sirve para:

Guardar el historial de cambios de un proyecto.

Volver a versiones anteriores.

Trabajar con ramas (branches).

GitHub es un servicio web que usa Git.
Proporciona:

Un lugar remoto donde guardar repositorios Git.

Herramientas de colaboración (issues, pull requests, revisiones).

Interfaz web para gestionar proyectos.


2.-Describe los tipos de repositorios que existen en GitHub:

En GitHub existen principalmente dos tipos de repositorios:

Repositorios públicos:
Son aquellos cuyo contenido puede ser visto por cualquier persona. Cualquier usuario puede clonar el repositorio y revisar el código. Son comunes en proyectos de software libre y de código abierto.

Repositorios privados:
Son repositorios cuyo contenido solo puede ser visto por el propietario y las personas autorizadas. Se utilizan para proyectos personales, académicos o empresariales donde el código debe mantenerse confidencial.

Además, los repositorios pueden pertenecer a un usuario individual o a una organización, lo que permite que varios colaboradores trabajen en un mismo proyecto con distintos niveles de permisos.
